What Makes a Good Prompt?

A good prompt consists of 5 sections/parts. Firstly, you need to give the AI context on how you want it to act, for example if you want it to give you stock advice say: You are a senior stock analyst, so you can set the tone and standard. Then you must outline the problem in detail and provide a clear outline of the solution. A common mistake people make is that they treat the AI as if it knows what you mean, however it is not the case and one must be precise about everything. Finally, you must mention the output format, for example a doc, a poster, etc.

Judgement

All 3 AIs were given the same prompt with no edits and in one response. Each game was scored out of 10 on 3 categories being: Quality of code, enjoyment, and adherence to the prompt. You can find the prompt below.

You are an expert game developer and creative designer. Your task is to create a complete, polished browser game contained entirely within one single HTML file. The result must be a fully playable game that runs when the user opens the .html file in Chrome. Do not split the project into multiple files — everything must be embedded in the same file.


Technical Constraints
The entire game must exist in one .html file. Use only HTML, CSS, and vanilla JavaScript. Do not import external libraries, CDNs, or assets. All graphics must be created using HTML5 Canvas, SVG, or CSS. The game must run completely offline with no internet connection. The code should be well-structured with clear sections and include comments explaining important parts. The game loop should run using requestAnimationFrame and maintain ~60 FPS.

Game Concept
Create a tower-defense style game where the player must protect a central core located in the middle of an open plane. Enemies spawn around the edges of the screen at random angles and attempt to reach the core. Unlike traditional tower-defense games, enemies must not follow preset paths — each enemy should calculate its own direct path toward the center.

Defense Units
Implement three different types: ① Sniper Tower — long range, single target, high damage. ② AOE Tower — short-to-moderate range, area-of-effect damage. ③ Support Tower — slows nearby enemies or buffs nearby towers. While placing a tower, display a dotted circle showing its attack range. Balance all three so no single tower dominates.

Game Rules & Progression
The player starts with a limited number of coins. Coins are spent to place towers. Destroying enemies rewards more coins. The central core starts with a fixed amount of health — enemies reaching it deal damage. The game ends when health reaches zero. Difficulty increases over time through faster spawn rates and faster enemy movement.

Scoring, UI & Visuals
Track and display: time survived, enemies destroyed, coins, and core health. Show a final score screen on game over. Include a tower selection panel with tooltips describing each tower. Add glow effects, animations, and visual attack indicators — lines, pulses, or circles — to make the game feel alive. Output format: a single, self-contained .html file.
